## Building a Bitcoin L1 Interface

Connecting a frontend to MidInt is very similar to standard EVM development, but with a few specific configurations to handle Bitcoin wallet signing.

### 1. Configuration

The core configuration for MidInt lives in `frontend/src/config/midl.ts`. We use `wagmi` and `viem` for blockchain interactions.

```typescript
import { midlRegtest } from '@midl/executor';

export const config = createConfig({
  chains: [midlRegtest],
  // ... connectors and providers
});
```

### 2. Wallet Connection

We recommend using `@midl/satoshi-kit` for the smoothest integration with Bitcoin wallets like Xverse and Unisat.

```tsx
import { SatoshiKitProvider, ConnectButton } from '@midl/satoshi-kit';

function App() {
  return (
    <SatoshiKitProvider>
      <ConnectButton />
      {/* Your app content */}
    </SatoshiKitProvider>
  );
}
```

### 3. Reading Data

You can use standard `wagmi` hooks to read data from your DEX contracts.

```tsx
const { data: balance } = useReadContract({
  address: TBTC_ADDRESS,
  abi: ERC20_ABI,
  functionName: 'balanceOf',
  args: [userAddress],
});
```

### 4. Executing Transactions

<Warning>
**Transaction Signing**: MidInt requires specific metadata for Bitcoin signing. When using `useWriteContract`, ensure the `WagmiMidlProvider` is wrapping your application to correctly intercept and sign intentions.
</Warning>

```tsx
const { writeContract } = useWriteContract();

const handleSwap = () => {
  writeContract({
    address: ROUTER_ADDRESS,
    abi: ROUTER_ABI,
    functionName: 'swapExactTokensForTokens',
    args: [amountIn, amountOutMin, path, to, deadline],
  });
};
```
